해당 웹 포트폴리오 및 프로젝트를 진행하면서 겪었던 문제들.

1. 깃헙의 액션기능을 몰라서 직접 개인 서버에 Docker를 통해 nginx로 서버를 로딩.
Bridge 네트워크 호스트를 통해 역방향 프록시로 개인 웹 도메인으로 포트폴리오 페이지를 오픈했는데
깃헙의 액션 기능에서 프로젝트 빌드 및 배포가 자동으로 가능하다는점을 몰랏기에 이런 번거로운 일을 했었다.

2. 깃헙의 액션기능은 생각보다 느리고 배포시 변경사항 적용이 안되는경우가 많다.
이유는 모르겟으나 현재도 이 레포지토리에서 돌아가고있는 액션은 2개다.
하나는 빌드되어서 정상배포되고있으나, 하나는 run cancle도 안먹히고, 
html 수정한뒤 리빌드후 배포를 하면 수정사항이 적용되지않고 예전코드로 돌려지는경우가 많아 불편했다.
빌드 - 배포 가 매끄럽고 자연스럽게 가능한점 - 좋았다. 
하지만 사양 수정 - 빌드 - 배포 까지는 지금까지 내 개인 NAS 서버에서 배포하듯이
로컬에서 테스트를 전부 진행한뒤 진행하는게 맞다는 생각이 든다.

3. 기본적인 css와 html의 문법 정립 및 가독성 있는 코드구조를 이해했다.
지금까지의 내 웹프로젝트는 hover 같은 애니메이션이나 UI UX 에 집중한 프로젝트가 아닌
기능 구현에만 집중한 ( 마이페이지, 찜, 1대1 채팅, 댓글 ) JSP 기반의 웹서버였다.
해당 JSP에 웹소켓 매니저등을 달아서 AI 한테 의존성 확인 등을 해가며 만든 웹프로젝트였으나.
문제는 코드의 가독성이 너무나도 좋지 못하다는것이다. (물론 그만큼 JSP는 그자체의 장단이 있다)
하지만 이번 순수 html, css, js 로 사이트를 구현하면서 전체적인 사이트의 구조를 개편해가며 만드는 방식을 배웟고.
웹사이트를 구축하며 마치 내가 좋아하는 취미 ( 프라모델 조립, 레고 조립 ) 같이 모듈을 하나하나 조립해가며 만들어가는 느낌이 들어 좋았다.

4. 마지막으로 마치며.
나는 원래 해당 강의에서 html과 css 기초를 다시 새겨가며 배울때 배울것이 없고 복습을 많이 할것같다는 생각을 하며 강의를 들었지만
수업을 진행함에 따라 내 기초가 얼마나 부족한지, 내가 지금까지 코딩/ 서버 가동을 얼마나 들이받기식으로 막무가내로 진행하였는지 다시한번 알게되었다.
이후에 js, nuxt, tailwind 등등의 최신 웹 도구들을 배우고 싶고, 백엔드에 더더욱 흥미가 생기게 되었다.
나는 내 서버를 내가 만든 웹 프로젝트들로 꽉 채워 가동할수 있게 되는날이 기대된다.
